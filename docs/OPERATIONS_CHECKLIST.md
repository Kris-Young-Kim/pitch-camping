# 운영 체크리스트

지속적인 서비스 운영을 위한 주기적 점검 체크리스트입니다.

## 일일 체크리스트

### API 상태 확인
- [ ] 고캠핑 API 응답 속도 확인 (< 2초 목표)
- [ ] 고캠핑 API 에러율 확인 (< 1% 목표)
- [ ] Rate Limit 이슈 확인
- [ ] API 키 유효성 확인

### 서비스 상태 확인
- [ ] Vercel 배포 상태 확인
- [ ] Supabase 데이터베이스 연결 상태 확인
- [ ] Clerk 인증 서비스 상태 확인
- [ ] 네이버 지도 API 사용량 확인

### 에러 모니터링
- [ ] Vercel 로그 확인 (에러 발생 여부)
- [ ] 관리자 대시보드 에러율 확인
- [ ] 사용자 피드백 확인 (긴급 이슈)

## 주간 체크리스트

### 성능 모니터링
- [ ] 페이지 로드 시간 확인 (LCP < 2.5초 목표)
- [ ] API 평균 응답 시간 확인
- [ ] 데이터베이스 쿼리 성능 확인
- [ ] Lighthouse 점수 확인 (80+ 목표)

### 사용자 지표
- [ ] 일일 활성 사용자 수 (DAU)
- [ ] 주간 활성 사용자 수 (WAU)
- [ ] 캠핑장 조회 수 추이
- [ ] 북마크 수 증가율
- [ ] 리뷰 작성 수

### 데이터 품질
- [ ] 캠핑장 데이터 정확도 확인
- [ ] 이미지 로드 실패율 확인
- [ ] 지도 마커 정확도 확인
- [ ] 검색 결과 관련성 확인

### 보안 확인
- [ ] 환경 변수 노출 여부 확인
- [ ] API 키 노출 여부 확인
- [ ] 데이터베이스 접근 로그 확인
- [ ] 의심스러운 활동 감지 여부

## 월간 체크리스트

### 비용 관리
- [ ] Vercel 사용량 및 비용 확인
- [ ] Supabase 사용량 및 비용 확인
- [ ] Clerk 사용량 및 비용 확인
- [ ] 네이버 지도 API 사용량 및 비용 확인
- [ ] 총 운영 비용 분석 및 예산 대비 확인
- [ ] 비용 증가 추세 분석

### 인프라 관리
- [ ] 데이터베이스 백업 확인
- [ ] 마이그레이션 파일 정리
- [ ] 불필요한 파일/데이터 정리
- [ ] 스토리지 사용량 확인 (Supabase Storage)
- [ ] CDN 캐시 상태 확인

### 기능 개선
- [ ] 사용자 피드백 분석 및 우선순위 정리
- [ ] 에러 로그 패턴 분석
- [ ] 성능 병목 지점 확인
- [ ] 기능 개선 제안 정리

### 데이터 분석
- [ ] 사용자 행동 패턴 분석
- [ ] 인기 캠핑장 분석
- [ ] 검색 키워드 분석
- [ ] 지역별 사용 패턴 분석

## 분기별 체크리스트

- [ ] 사용자 만족도 평가 (피드백 기반)

### 기술 부채 관리
- [ ] 코드 리뷰 및 리팩토링 계획
- [ ] 의존성 업데이트 검토
- [ ] 보안 취약점 점검
- [ ] 성능 최적화 기회 발굴

### 로드맵 검토
- [ ] 다음 분기 개발 계획 수립
- [ ] 우선순위 재조정
- [ ] 리소스 배분 계획

## 비상 대응 체크리스트

### 서비스 중단 시
1. [ ] Vercel Status 확인
2. [ ] Supabase Status 확인
3. [ ] 외부 API 상태 확인 (고캠핑 API, 네이버 지도 API)
4. [ ] 에러 로그 확인
5. [ ] 롤백 필요 여부 판단
6. [ ] 사용자 공지 (필요시)

### 데이터 손실 시
1. [ ] 최근 백업 확인
2. [ ] 데이터 복구 절차 실행
3. [ ] 손실 범위 파악
4. [ ] 재발 방지 대책 수립

### 보안 이슈 발생 시
1. [ ] 문제 범위 파악
2. [ ] 영향받는 사용자 확인
3. [ ] 즉시 조치 (비밀번호 변경, API 키 재발급 등)
4. [ ] 사용자 공지 (필요시)
5. [ ] 사후 조치 계획 수립

## 알림 설정

### 임계값 기반 알림
- [ ] API 응답 시간 > 3초
- [ ] 에러율 > 5%
- [ ] 페이지 로드 시간 > 5초
- [ ] 데이터베이스 쿼리 시간 > 1초
- [ ] 비용 한도 초과 (월간 예산의 80%)

### 알림 채널
- [ ] 이메일 알림
- [ ] Slack/Discord 알림 (선택 사항)
- [ ] SMS 알림 (긴급 이슈, 선택 사항)

## 체크리스트 사용 방법

1. **일일 체크리스트**: 매일 아침 또는 저녁에 확인
2. **주간 체크리스트**: 매주 월요일에 확인
3. **월간 체크리스트**: 매월 첫째 주에 확인
4. **분기별 체크리스트**: 분기 첫째 주에 확인
5. **비상 대응 체크리스트**: 문제 발생 시 즉시 확인
